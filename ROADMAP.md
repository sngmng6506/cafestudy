# Roadmap

아래 항목들은 앞으로 추가되면 좋은 기능 후보입니다. 관심 있는 사람이 원하는 항목을 골라 구현해도 됩니다.

## 구현 후보

### 카페 정보
- 모임이 열린 적 있는 카페 목록을 보여준다.
- 카페 이름, 위치, 방문 이력, 관련 모임을 함께 확인할 수 있게 한다.
- 네이버 장소 검색 결과와 연결해 카페 식별을 더 안정적으로 만든다.

### 카페 이력과 코멘트
- 완료된 모임의 카페를 카페 이력에 남긴다.
- 해당 카페에 참석한 적 있는 사람만 한줄 코멘트를 남길 수 있게 한다.
- 같은 카페가 다른 문자열로 쪼개지지 않도록 장소 ID 또는 정규화된 키를 도입한다.

### 모임 이력
- 시간이 지나 완료된 모임을 모임 이력 탭으로 이동시킨다.
- 완료 시점의 참석자 목록을 보존한다.
- 나중에 인증 사진이 올라오면 해당 모임 이력에 연결해 보여준다.

### 지도 연동
- 앱의 지도보기 버튼에 해당하는 장소 정보를 웹에서도 확인할 수 있게 한다.
- 웹 크롤링만으로 장소 좌표를 얻을 수 있는지 추가 확인한다.
- 불가능하면 장소명 기반 검색 API로 보완한다.

### 앱 자동화 API
- 웹에 없는 운영진 기능을 앱 자동화로 조작할 수 있는지 검토한다.
- 후보 기능: 모임 개설, 모임 수정, 참석자 확인.
- 에뮬레이터, 실기기, 미니 PC 중 어떤 운영 방식이 현실적인지 결정한다.

### 모임 정산
- 완료된 앱 모임에 정산을 연결한다.
- 모임 개설자가 총액과 참가자별 금액을 등록한다.
- 참가자는 자신의 금액과 입금 상태를 확인한다.
- 초기 버전은 균등 분할과 입금 확인에 집중하고, 항목별 비용·차등 분배는 이후 확장한다.

## 구현됨: 공지사항·알림·관리자 권한

현재 구현:

- 모든 로그인 사용자가 공지사항 목록을 확인할 수 있다.
- 우상단 종 아이콘에 읽지 않은 공지 개수가 표시된다.
- 종을 누르면 최근 공지를 확인하고 개별 공지 또는 전체 공지를 읽음 처리할 수 있다.
- 읽음 상태는 사용자별 PostgreSQL 기록으로 저장돼 다른 기기에서도 유지된다.
- `owner`, `admin`, `member` 3단계 역할을 사용한다.
- 이상명 계정은 최고 관리자 `owner`로 지정된다.
- `owner`만 다른 멤버를 관리자로 임명하거나 관리자 권한을 해제할 수 있다.
- `owner`와 `admin`은 관리자 페이지에서 공지를 작성·수정·삭제·상단 고정할 수 있다.
- 역할 변경은 `admin_role_logs`에 변경 전후 역할과 변경자를 기록한다.
- 기존 `users.is_admin`은 호환성을 위해 `admin_role`과 함께 유지한다.

## 구현됨: 자연어 메뉴 탐색

현재 구현:

- 탭바 바로 위의 전역 검색 캡슐에서 한국어 자연어 질의를 입력한다.
- 메뉴 설명·키워드·예시 문장을 검색 대상으로 사용한다.
- 한국어 키워드 점수 70%와 `@ternlight/mini` 의미 점수 30%를 결합한다.
- Ternlight는 Vue 클라이언트의 브라우저에서 WASM으로 실행하며 별도 Railway 모델 서버를 두지 않는다.
- 메뉴 문장은 최초 의미 검색 때 임베딩하고 브라우저 메모리에 캐시한다.
- 메뉴 수가 적으므로 벡터 DB 없이 코사인 유사도를 직접 비교한다.
- 모델 로딩이나 실행에 실패하면 한국어 키워드 검색만 사용한다.
- 검색 결과 상위 3개를 표시하고 선택하면 기존 기능 화면으로 이동한다.
- 더보기의 `검색 안내` 화면에서 동작 방식과 제한을 확인할 수 있다.

### 검색 품질 개선 후보

- 실제 한국어 검색 질의 30~50개를 테스트 세트로 만들고 Top-1·Top-3 정확도를 측정한다.
- 메뉴별 설명, 검색어와 예시 문장을 실제 오답 사례에 맞춰 보강한다.
- 영어 중심 Ternlight의 한국어 의미 점수가 실제로 순위 개선에 기여하는지 비교한다.
- 기여가 낮거나 오답을 늘리면 의미 점수 가중치를 더 낮추거나 키워드 동점 보정에만 사용한다.
- 모바일에서 최초 WASM 다운로드·초기화 시간과 메모리 사용량을 측정한다.
- 메뉴가 크게 늘어나거나 문서 검색으로 확장될 때만 다국어 서버 임베딩 또는 사전 계산 벡터를 검토한다.
